# Contributing to MCU-Debug

Thank you for your interest in improving MCU-Debug! To protect the project and its users, we require all contributors to adhere to the following guidelines.

## 1. Licensing Agreement
By contributing to this repository, you agree that:
* Contributions to the **VS Code Extension (TypeScript/DAP)** are licensed under the **MIT License**.
* Contributions to the **MCU-Debug Helper (Rust)** are licensed under the **Apache License, Version 2.0**.
* You represent that you are legally entitled to grant these licenses and that your contribution does not infringe on any third-party patents or copyrights.

## 2. Corporate Contributions
If you are contributing on behalf of a corporation (e.g., ARM, ST, Infineon), please ensure you have the authority to grant the **Apache 2.0 Patent License** included in the Rust modules's licensing terms. 

## 3. Code Style & Standards
* **Rust:** All code must be formatted with `rustfmt` and pass `clippy` without warnings.
* **TypeScript:** Follow existing ESLint/Prettier configurations.
* **Documentation:** Updates to logic must be reflected in the relevant README or inline documentation.

## 4. How to Submit
1.  Fork the repository.
2.  Create a feature branch.
3.  Submit a Pull Request (PR) with a clear description of the changes.
4.  By submitting the PR, you implicitly sign off on the Licensing Agreement stated above.
